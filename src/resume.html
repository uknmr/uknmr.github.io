仕事ではウェブサービスのフロントエンドエンジニアとして設計や開発に携わることが多いです。
プロジェクトではサービスのドメインオーナーと話し合いながら（時にはオーナーとして）実現方法を提案・設計から実装まで行っていました。
The Great Divide の二極で表すと JavaScript スペシャリストというよりは UX エンジニアです。

ユーザー利益を追い求めることが第一優先であると考えた上で、開発環境を改善する FrontendOps と呼ばれる領域に携わることも多いです。
具体的には CI/CD の環境構築による自動化や開発を補助するライブラリやツールの作成、パフォーマンス計測の自動化や改善、アクセシビリティの担保などを行っていました。
テックリードとして組織戦略を考えモダンな環境を目指したこともありますが、どちらかといえば運用しやすく捨てやすい開発環境を作ることを目指しています。

また同時にデザイナーとしてもチームの力を最大化するために複数ワークショップを行い、その中からサービス理念を作り上げたり、デザインシステムを作り上げたりしました。
デザインへの理解が乏しい組織では「デザイン」という言葉の再定義を行い、デザイン批評の教育から仕組みづくりまで行いました。

最近の第一言語は TypeScript です。React / Redux や Vue を用いてアプリケーション開発に関わることもありましたが、プロジェクトの状況を鑑みて VanillaJS に戻すことも同じくらいありました。
過去にはサーバサイド言語を第一言語として DB 設計や API 設計、実装を行っていたこともありますが、もう 5 年近く離れているためできないといっても過言ではありません。

やりたいこと

ウェブデザインの多くは狭義に捉えられることが多くチームとしてどうユーザーに価値を届けるのか、というところまでやりきれている組織は少ないです。それを実現するにはウェブやクライアントサイドとデザインを理解した上でビジネスやシステムとつなぐ役割が必要であると考えています。

IA

人が抱える課題や混乱を解決したり紐解いたりすることが広義の IA でありデザインであると捉えています。

ユーザーへの提供速度を保ちつつも、ユーザー利益を追い求め、長い目で見てブランドを資産に仕立てあげるような仕事に興味があります。

インターネット

インターネットが好きなことを自負していましたが、広義の IA やアクセシビリティといった考え方を通し改めて「人類の可能性としてのインターネット」が好きなことに最近気が付きました。ウェブが誕生して 30 年立ちましたが、セマンティック・ウェブをまだ信じています。
